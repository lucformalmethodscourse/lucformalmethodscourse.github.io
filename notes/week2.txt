Software Supply Chain and Engineering

Static vs Dynamic Non-Functional Requirements
Nicholas discussed the importance of distinguishing between static and dynamic non-functional requirements in software development. He explained that static non-functional requirements set standards, while dynamic non-functional requirements focus on performance and usability. He used the example of a web server that needs to handle at least 10,000 simultaneous users as a dynamic non-functional requirement. Nicholas also demonstrated how to create a project in a terminal, using Python to build a web server that hosts a simple web page. He emphasized that modern software engineering is not about writing every line of code from scratch, but about setting requirements and using tools like Chat GPT to generate code.
Python Software Dependencies and Management
Nicholas discussed the use of software dependencies in Python programming, highlighting the role of the software supply chain in reusing code. He explained the process of installing a package called Aio Http and its dependencies, and how it can be used to handle a million connections. He also touched on the concept of dependency management, using the example of a web server that depends on Python, Aio Http, and Pandas. He demonstrated how to analyze dependencies and their transitive dependencies, emphasizing the importance of version management and the use of build lifecycle tools. He also mentioned the use of tools like Maven and Poetry for building projects.
Software Supply Chain Vulnerabilities
Nicholas discussed the vulnerabilities in software supply chains, specifically focusing on the "left pad" and "log4j" incidents. He explained how these vulnerabilities could have been mitigated with formal specifications, allowing for language-agnostic implementations. He also highlighted the importance of dependency management and the potential use of large language models for formal verification of dependencies. Lastly, he introduced the concept of Software Bills of Materials (SBOMs) and the tool Trivia for security auditing.

Exploring Propositional and Predicate Logic
Konstantin discussed propositional logic, explaining concepts such as conjunction, negation, and implication. He used the example of "if it's sunny outside, then we'll go for a walk" to illustrate how implications work. He then introduced predicate logic, which involves quantifiers like "for all" and "exists". He used the example of "for all X, if X is a cat, then P of X holds" to demonstrate how universal quantification works. He emphasized the importance of specifying the domain of the variable in predicate logic. He also touched on the idea of applying universal quantification to more complex properties, including implications.
Predicate Logic and Proof Checkers
Konstantin discussed the intricacies of predicate logic, focusing on universal quantification and negation. He explained how negating a universally quantified statement results in the existence of at least one instance that does not satisfy the statement. Konstantin also introduced the concept of proof checkers, which can rigorously check logic statements and proofs. He demonstrated how to use a proof checker to verify the correctness of an implementation of list reversal. The discussion concluded with Konstantin explaining the process of proof by induction, using the example of list reversal.
